# Three-Minute Demo Script

## 0:00–0:25 — Set the frame

“Diligence Readiness Layer is an independent feature prototype for an operator helping founders prepare evidence for fundraising. It runs locally, uses deterministic rules, and treats every generated output as a draft for review.”

Open `/demo` and select **Seed all companies**.

## 0:25–0:55 — Show the portfolio

Open `/companies`.

Point out that five synthetic companies have different stages, scores, tiers, risks, and review states. Open AtlasAI, then FinPilot, to show that company routes preserve distinct evidence.

## 0:55–1:30 — Explain the honest score

On AtlasAI's dashboard, show:

- Strict Raise Readiness Score: 53.4
- Tier: Not diligence-ready
- Recovery path: approximately 78–84
- weighted component breakdown

Explain that the prototype preserves the strict baseline and shows improvement separately.

## 1:30–2:05 — Trace evidence into work

Open AtlasAI's diligence concerns and investor Q&A. Show source references, missing evidence, investor relevance, operator notes, and estimated score lift.

Open Confidence Audit and point out all six components plus the explicit heuristic-estimate caveat.

## 2:05–2:40 — Create operator-entered data

Open **Intake company**, create a company, and go to **Edit data**.

Add:

- one financial record;
- one compliance item;
- one cap-table holder with **Founder / founding holder** checked;
- one text note.

Run analysis and show the partial dashboard.

## 2:40–3:00 — Export and close

Export the Markdown diligence report.

Close with: “The prototype does not replace advisers or operator judgment. It makes missing evidence, deterministic calculations, and the preparation queue easier to inspect.”
